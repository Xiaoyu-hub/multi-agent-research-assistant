# Multi-Agent Research Assistant

> 基于 LangChain 的多 Agent 协作研究助手，支持 RAG 检索、任务规划与协同写作

## 项目概述

### 项目目标

本项目旨在构建一个智能的多 Agent 协作系统，通过专业化分工和协同工作，高效完成复杂的研究任务。系统能够自动分解任务、搜集信息、整合分析并生成高质量的研究报告。

### 应用场景

- **学术研究**：快速调研特定领域的最新进展和研究趋势
- **行业分析**：收集市场信息、技术动态和竞争情报
- **技术评估**：评估新技术的应用前景和可行性
- **知识管理**：整合企业内部知识库和外部信息源

### 系统价值

- **提高效率**：自动化完成信息搜集和分析过程，减少人工工作
- **增强深度**：多视角分析，提供更全面、深入的研究结果
- **保证质量**：通过多轮审核和反馈机制，确保输出内容的准确性和可靠性
- **可扩展性**：模块化设计，易于添加新的 Agent 类型和功能

## 设计原则

- **模块化**：各 Agent 职责明确，独立可扩展
- **协作性**：通过 Planner Agent 实现 Agent 间的有效协作
- **可靠性**：单个 Agent 失败不影响整体系统运行
- **可追溯性**：完整记录各 Agent 的执行过程和结果
- **可配置性**：支持通过配置文件调整系统参数和行为

## Agent 角色定义

### 1. Planner Agent

**职责**：任务分解与调度

```
输入: 用户问题
处理:
  1. 分析问题类型
  2. 拆解为子任务列表
  3. 分配给专业 Agent
  4. 收集结果并整合
输出: 整合后的完整报告
```

**Prompt 设计要点**：

- 明确任务边界
- 定义子任务描述
- 指定执行顺序（并行/串行）

### 2. Search Agent × 3

**职责**：从不同角度搜集信息

| Agent                 | 搜索角度 | 数据源           |
| --------------------- | ---- | ------------- |
| Search-Agent-Paper    | 学术论文 | arXiv / 论文库   |
| Search-Agent-Industry | 工业应用 | 新闻 / 博客       |
| Search-Agent-Tech     | 技术突破 | 官方文档 / GitHub |

**共用工具**：

- DuckDuckGo Search
- Web Fetch

### 3. RAG Agent

**职责**：基于私有知识库回答

**底层架构**：

```
文档加载 → 分块 → Embedding (bge-large-zh-v1.5) → FAISS 索引 → 检索 → LLM 生成
```

**工具**：

- VectorStore Retriever
- Web Search (补充)

### 4. Writer Agent

**职责**：整合多源信息，生成结构化报告

### 5. Critic Agent

**职责**：质量审核（Reflection 模式）

```
输入: 报告草稿
判断:
  - 信息是否完整?
  - 逻辑是否一致?
  - 是否有幻觉?
输出:
  - 通过 → 返回最终报告
  - 不通过 → 指出问题 → 打回 Writer 修改
```

## 技术栈

| 层次        | 技术                               | 说明                 |
| --------- | -------------------------------- | ------------------ |
| Agent 框架  | LangChain                        | Agent 定义、工具注册、链式调用 |
| LLM       | gpt-4o-mini / qwen-max           | 生成能力               |
| RAG       | LangChain + FAISS + bge-large-zh | 知识库检索              |
| 搜索        | DuckDuckGo Search                | 实时信息检索             |
| Embedding | BAAI/bge-large-zh-v1.5           | 向量化模型              |
| 向量库       | FAISS                            | 本地向量索引             |
| Memory    | ConversationBufferMemory         | 多轮上下文              |
| API 服务    | FastAPI                          | 对外提供服务             |

## 面试可展开的话题

1. **Agent 设计**：5 个 Agent 的角色定义和 Prompt 设计思路
2. **Planner 如何拆解任务**：树结构 vs 列表，串行 vs 并行
3. **Memory 隔离**：每个 Agent 独立 Memory vs 共享 Memory
4. **RAG 优化**：chunk 策略、Embedding 选型、MMR、混合检索
5. **Critic 的自检逻辑**：Reflection 模式实现
6. **错误处理**：单个 Agent 失败不影响整体流程
